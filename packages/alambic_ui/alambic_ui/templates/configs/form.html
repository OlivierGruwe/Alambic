À quoi ça sert
  • Ces champs sont attachés au dépôt et hérités par tous les documents de la transaction — y compris les pièces qui ne portent pas ces informations elles-mêmes.
  • Information mail : récupère une donnée de l'e-mail source (expéditeur, objet, date). Utile quand le dépôt vient d'un e-mail.
  • Calculé : valeur générée au traitement (@today, @now, @year_month, @transaction_id, @config_name).
Comment ça marche, étape par étape
  1. À l'arrivée d'un dépôt, chaque champ propagé est résolu une fois pour la transaction.
  2. Information mail : la valeur est lue dans les métadonnées de la source (ex. from → l'adresse de l'expéditeur). Si la source ne fournit pas la donnée, la valeur par défaut est utilisée.
  3. Calculé : la valeur est générée (ex. @today → 2026_06_30).
  4. La valeur résolue est ajoutée comme index à chaque document de la transaction, sous le nom du champ. Les sous-documents issus d'un découpage en héritent aussi.

À quoi ça sert
  • Définir des web services externes appelés pour valider ou enrichir un champ extrait (ex. vérifier un numéro de compte et récupérer le titulaire).
  • Chaque WS cible un champ précis d'un doctype attendu, choisi dans la colonne « Champ cible » (format @doctype:champ). Tout se règle ici, rien à configurer côté Types de documents.
  • Sécurité : seuls les domaines autorisés au niveau du compte sont appelables, le secret est chiffré. En cas d'échec, le réglage « Si échec » décide (ignorer = non bloquant, par défaut).
Comment ça marche, étape par étape
  1. L'extraction trouve la valeur d'un champ (ex. numero_compte = "FR7612345").
  2. Si ce champ référence un WS, Alambic appelle l'URL en remplaçant {value} par la valeur extraite : https://api.banque.fr/comptes/FR7612345.
  3. Le WS répond en JSON, ex. { "exists": true, "titulaire": "Dupont", "agence": "Lyon 3e" }.
  4. La clé statut (ici exists) indique si la donnée est valide → un index numero_compte__consolidation_status vaut VALID, INVALID ou ERROR.
  5. Le mapping réponse ajoute les données reçues au document comme nouveaux index. titulaire:compte_titulaire crée un index compte_titulaire = "Dupont".
